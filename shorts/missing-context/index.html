The mindshift

Claude has no memory.
They give Claude the memory.
They just have to keep writing it down.

Here's the thing

This is the same instinct they've always had.

design docsRFCscode review briefsarchitecture decision recordsonboarding guidestickets with real acceptance criteria

They've been writing things down for collaborators who can't read their minds their whole careers. Every one of those is a blueprint. A way of saying what they want clearly enough that someone else can do the work.

Claude needs exactly that. In a file that survives the chat closing.

Nobody has the reps for it yet with this collaborator. They spent years getting good at writing the code themselves. That's why it feels slow. Unfamiliar work always does.

Here's the part I want to be honest about

Some of us are thinking it: "who wants to edit AI-generated code?"

Fair.Real.Correct, even.

Fixing the same half-wrong output twice, explaining the same conventions to a window that forgets everything between sessions, losing the thread of thought to a cascade of small corrections. That is genuinely exhausting work. We shouldn't be doing it.

Notice: they're not just fixing code. They wrote the rules that will pay off in the future.

That's the move for us too. The blueprint is what we're building. Claude's output is how we find out which rule is still missing. Each round builds the muscle memory so we need fewer corrections over time.

The shape of the week, before and after

This is where the compounding starts.

Every minute they put into the blueprint is a minute Claude won't need them to repeat tomorrow. Every rule they write once carries for the rest of the project.

Some tasks slide. Gnarly bugs, perf work, regulated paths, anything with deep tacit knowledge. The ratio shifts. Most of your week still fits this shape. The hard parts are what the workshop is for.

The part that feels like cheating

Eventually they stop editing the blueprint themselves.

The rules file grows into a directory: .claude/rules/. One file per concern. Styling. Data. Code standards. Self-correction. Claude reads the right one when it needs it, not all of them at once.

Every correction becomes a permanent rule in the right file. Claude maintains its own context, spoon-fed on demand. The rules compound.

When the team aligns on what belongs in the rules, when we've built the muscle memory for what to write down, the execution becomes something we can trust. That's when the planning stops feeling like overhead and starts feeling like the part that's ours.

The planning, designing, and delegating becomes the work.
